## 三、Ollama 离线模式 —— 推荐轻量模型

### 3.1 已部署：deepseek-r1:7b（推理对话首选）

```bash
# 已部署，直接使用
ollama run deepseek-r1:7b
```

 **适用场景** ：专利问答对话（ChatService）、专利技术解析

### 3.2 推荐补充部署的模型

| 用途                            | 模型名称             | 显存要求 | 拉取命令                         | 说明                           |
| ------------------------------- | -------------------- | -------- | -------------------------------- | ------------------------------ |
| **实体提取/分类（主力）** | `qwen2.5:7b`       | ~5GB     | `ollama pull qwen2.5:7b`       | 中文理解最佳，现有配置默认模型 |
| **轻量备用（低显存）**    | `qwen2.5:3b`       | ~2.5GB   | `ollama pull qwen2.5:3b`       | 资源受限时替代 7b              |
| **推理增强**              | `deepseek-r1:7b`   | ~5GB     | 已部署                           | 复杂匹配评估场景               |
| **文本向量化（必须）**    | `nomic-embed-text` | ~300MB   | `ollama pull nomic-embed-text` | 替代在线 Embedding，768维      |
| **备用 Embedding**        | `bge-m3`           | ~600MB   | `ollama pull bge-m3`           | 中文向量化效果更好，1024维     |

### 3.3 模型功能分工建议

```txt
LlmService（实体提取 + 匹配评估）  →  qwen2.5:7b  或  deepseek-r1:7b
ChatService（专利问答对话）         →  deepseek-r1:7b（推理能力强）
VectorService（向量化 Embedding）   →  nomic-embed-text  或  bge-m3
```

## 四、在线模式 —— OpenAI 兼容 API 方案

### 4.1 两级 API 配置机制

**第一级：系统预设（环境变量，管理员配置）**

```yaml
# application.yml - 系统默认在线API
spring:
  ai:
    openai:
      base-url: ${ONLINE_API_BASE_URL:https://dashscope.aliyuncs.com/compatible-mode}
      api-key: ${ONLINE_API_KEY:}
      chat:
        options:
          model: ${ONLINE_CHAT_MODEL:qwen-plus}
      embedding:
        options:
          model: ${ONLINE_EMBEDDING_MODEL:text-embedding-v3}
```

**第二级：用户自定义（数据库持久化，用户个人设置）**

新增 `sys_llm_config` 表：

```sql
CREATE TABLE sys_llm_config (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id     BIGINT NOT NULL COMMENT '0=系统默认, >0=用户自定义',
    config_name VARCHAR(64) NOT NULL COMMENT '配置名称(如 GPT-4o、通义千问)',
    llm_mode    VARCHAR(16) NOT NULL COMMENT 'online/offline',
    base_url    VARCHAR(256) COMMENT '自定义 API BaseURL',
    api_key     VARCHAR(512) COMMENT '加密存储的 API Key',
    chat_model  VARCHAR(128) COMMENT '对话模型名称',
    embed_model VARCHAR(128) COMMENT '向量模型名称',
    is_active   TINYINT DEFAULT 0 COMMENT '是否当前启用',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_user_active (user_id, is_active)
);
```

## 七、向量维度兼容问题（重要！）

不同 Embedding 模型输出维度不同，Qdrant Collection 创建后维度固定：

| 模型                              | 维度 | 切换代价                 |
| --------------------------------- | ---- | ------------------------ |
| `text-embedding-v3`（通义千问） | 1024 | 需重建 Collection        |
| `nomic-embed-text`（Ollama）    | 768  | 需重建 Collection        |
| `bge-m3`（Ollama）              | 1024 | 与通义千问兼容           |
| `deepseek` 无官方 Embedding     | -    | 用通义千问或 bge-m3 替代 |

 **建议** ：统一选用 **`bge-m3`（离线）** 和 **`text-embedding-v3`（在线）** 均为 1024 维，模式切换无需重建向量库。

## 在线模式三模型分工 —— 完全可行，且推荐这样做

### 可行性说明

### [`LlmConfig.java`](vscode-webview://0sib8vp78qnf5h97hfd70jmf9b2k2a40jcug72lsbjtaj2d8id37/backend/src/main/java/com/patent/config/LlmConfig.java) 改造 —— 注册三个独立 Bean

```java
@Slf4j
@Configuration
public class LlmConfig {

    /**
     * 对话 ChatModel（ChatService 使用）
     * online → qwen-max，offline → deepseek-r1:7b
     */
    @Bean("chatChatModel")
    @Primary
    public ChatModel chatChatModel(
            OpenAiChatModel openAiChatModel,
            OllamaChatModel ollamaChatModel,
            PatentConfig config) {
        if ("offline".equals(config.getLlmMode())) {
            // 通过 Options 动态指定 deepseek-r1:7b
            return buildOllamaChat(ollamaChatModel, config.getOllama().getChatModel());
        }
        return buildOpenAiChat(openAiChatModel, config.getOnline().getChatModel());
    }

    /**
     * 分析 ChatModel（LlmService 使用 - 实体提取、匹配评估）
     * online → qwen-plus，offline → qwen2.5:7b
     */
    @Bean("llmChatModel")
    public ChatModel llmChatModel(
            OpenAiChatModel openAiChatModel,
            OllamaChatModel ollamaChatModel,
            PatentConfig config) {
        if ("offline".equals(config.getLlmMode())) {
            return buildOllamaChat(ollamaChatModel, config.getOllama().getLlmModel());
        }
        return buildOpenAiChat(openAiChatModel, config.getOnline().getLlmModel());
    }

    /**
     * Embedding Model（VectorService 使用）
     * online → text-embedding-v3，offline → nomic-embed-text
     */
    @Bean("primaryEmbeddingModel")
    @Primary
    public EmbeddingModel primaryEmbeddingModel(
            OpenAiEmbeddingModel openAiEmbeddingModel,
            OllamaEmbeddingModel ollamaEmbeddingModel,
            PatentConfig config) {
        if ("offline".equals(config.getLlmMode())) {
            // Ollama embedding 模型固定为配置的 embedModel
            return ollamaEmbeddingModel;
        }
        return openAiEmbeddingModel;
    }
}
```

| 场景                     | 提供商   | chatModel               | llmModel                | embedModel               | 月成本参考  |
| ------------------------ | -------- | ----------------------- | ----------------------- | ------------------------ | ----------- |
| **生产（低成本）** | 通义千问 | `qwen-plus`           | `qwen-turbo`          | `text-embedding-v3`    | ~¥50/万次  |
| **生产（高质量）** | 通义千问 | `qwen-max`            | `qwen-plus`           | `text-embedding-v3`    | ~¥200/万次 |
| **DeepSeek 优先**  | DeepSeek | `deepseek-chat`       | `deepseek-chat`       | *需配合其他 Embedding* | ~¥20/万次  |
| **零成本测试**     | 硅基流动 | `Qwen2.5-7B-Instruct` | `Qwen2.5-7B-Instruct` | `bge-m3`               | 免费        |

> ⚠️  **注意** ：DeepSeek 官方 API 目前 **没有 Embedding 接口** ，若在线模式使用 DeepSeek 做对话/分析，Embedding 建议使用通义千问的 `text-embedding-v3` 或 Ollama 的 `bge-m3`（1024维，与在线模式维度兼容）。

### 向量维度统一建议（避免切换重建问题）

```txt
离线 Embedding：bge-m3          → 1024 维
在线 Embedding：text-embedding-v3 → 1024 维
✅ 两者维度一致，在线/离线随时切换，无需重建 Qdrant Collection
```

修改 [`application.yml`](vscode-webview://0sib8vp78qnf5h97hfd70jmf9b2k2a40jcug72lsbjtaj2d8id37/backend/src/main/resources/application.yml)：

```yaml
spring:
  ai:
    ollama:
      embedding:
        options:
          model: bge-m3   # 由 nomic-embed-text(768维) 改为 bge-m3(1024维)

patent:
  vector-dimension: 1024   # 统一1024维
```

这样 **在线模式和离线模式可以无缝切换，无需重建向量库** 。
