## 1. 选型原则

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **版本兼容性** | 确保各组件版本相互兼容，避免依赖冲突 |
| **开源友好性** | 优先选择开源、社区活跃的技术 |
| **部署便捷性** | 支持Docker容器化，一键部署 |
| **LLM集成能力** | 原生支持主流LLM（在线+离线） |
| **中文支持** | 良好的中文处理能力（分词、NLP） |

### 1.2 适配数据特点

根据专利数据特征进行技术选型：

| 数据特征 | 技术适配 |
|----------|----------|
| 中文专利文本（标题、摘要、权利要求） | IK中文分词器、通义千问embedding |
| IPC分类号（如G06F16/30） | ES Keyword字段 + Qdrant payload过滤 |
| PDF专利文档 | Apache PDFBox / Spring AI PDF Reader |
| 大规模数据（10万+专利） | ES分片 + Qdrant HNSW索引 |

## 6. AI与LLM层

### 6.1 在线模式（推荐生产环境）

| 组件 | 选型 | 说明 |
|------|------|------|
| **Chat模型** | 通义千问 qwen-plus / qwen-turbo | 阿里云，国内访问快 |
| **Embedding模型** | 通义千问 text-embedding-v3 | 中文优化，1536维 |
| **备选** | OpenAI gpt-4o-mini | 海外用户 |

### 6.2 离线模式（本地部署）

| 组件 | 选型 | 说明 |
|------|------|------|
| **LLM运行时** | Ollama | 本地模型运行环境 |
| **Chat模型** | Qwen2.5:7b | 阿里开源，中文能力强 |
| **Embedding模型** | nomic-embed-text / bge-m3 | 开源embedding |

### 6.3 Spring AI集成

```yaml
# application.yml 双模式配置
spring:
  ai:
    # 通义千问配置（在线模式）
    dashscope:
      api-key: ${DASHSCOPE_API_KEY}
      chat:
        options:
          model: qwen-plus
      embedding:
        options:
          model: text-embedding-v3
    
    # Ollama配置（离线模式）
    ollama:
      base-url: http://localhost:11434
      chat:
        options:
          model: qwen2.5:7b
      embedding:
        options:
          model: nomic-embed-text
    
    # Qdrant向量存储
    vectorstore:
      qdrant:
        host: localhost
        port: 6334
        collection-name: patent_vectors
        initialize-schema: true
```
