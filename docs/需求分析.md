## 1. 项目背景

### 1.1 研究背景

随着全球专利申请量的持续增长，专利技术匹配已成为科研人员、企业知识产权部门和技术转移机构面临的重要挑战。传统的专利检索方法主要依赖关键词匹配和分类号检索，存在以下问题：

- **语义理解不足**：无法准确理解专利文本中的技术概念和创新点
- **实体识别困难**：难以自动提取专利中的关键技术实体（如产品名称、技术方法、材料等）
- **领域关联缺失**：无法建立专利技术与层次化领域知识的关联
- **匹配精度有限**：传统方法难以实现语义级别的精准匹配

### 1.2 研究意义

本系统旨在利用大语言模型（LLM）的强大语义理解能力，结合实体识别和领域知识增强技术，构建一个智能化的专利技术匹配系统。该系统将：

1. 提升专利检索的准确性和效率
2. 实现专利文本中实体和领域信息的自动提取
3. 基于知识增强的方式提升技术匹配能力
4. 为用户提供直观的可视化分析界面

### 1.3 项目目标

根据毕业设计要求，本系统需实现以下核心目标：

| 序号 | 目标描述                                                                |
| ---- | ----------------------------------------------------------------------- |
| 1    | 构建大语言模型的运行环境，支持主流LLM的访问，涵盖在线和离线两种模式     |
| 2    | 设计大语言模型引导指令，用于挖掘专利文本中的实体和层次化领域信息        |
| 3    | 基于识别的实体和领域信息来增强大语言模型进行专利技术匹配的能力          |
| 4    | 采用B/S模式实现系统，支持用户进行专利技术信息的录入、检索和可视化等功能 |

### 2.2 竞品分析

#### 2.2.1 智慧芽（PatSnap）

**产品概述**：智慧芽是全球领先的专利情报平台，运用AI技术和深加工专利数据提供专业的全球专利情报服务。

**核心功能**：

- 多维度数据处理和深加工
- AI驱动的专利检索和筛选
- 专利解读和分析
- 协同工作空间
- 竞品监控与预警机制

**技术特点**：

- 覆盖全球主要专利局数据
- 支持中英文双语检索
- 提供专利价值评估
- 可视化分析报告

**优势**：

- 数据覆盖广泛
- AI功能成熟
- 用户界面友好

**不足**：

- 商业化产品，成本较高
- 定制化程度有限
- 缺乏开放的LLM集成接口

#### 2.2.2 incoPat（合享智慧）

**产品概述**：incoPat是全球专利数据库，提供可信、易用的专利检索和分析服务。

**核心功能**：

- **AI Search**：专利DNA图谱匹配，支持自然语言输入
- **语义检索2.0**：更精准的专利主题意图理解
- **智能检索策略**：基于大模型的检索式自动构建
- **图像检索**：支持结构图、流程图等图像搜索发明专利
- **多维度分析**：130+维度的自定义统计分析

**技术特点**：

- 收录173个国家/地区的2亿+专利数据
- 400+检索字段
- 中英文全文翻译
- 每周四次数据更新

**优势**：

- 数据质量高，更新及时
- AI搜索功能先进
- 支持DWPI（德温特全球专利索引）

**不足**：

- 主要面向企业用户
- 缺乏开源组件
- LLM集成能力有限

#### 2.2.3 Derwent Innovation（科睿唯安）

**产品概述**：Derwent Innovation是专业的专利研究应用，提供可信的专利数据和工具。

**核心功能**：

- 专利可专利性分析
- 自由实施（FTO）分析
- 有效性评估
- AI驱动的专利搜索

**技术特点**：

- 整合Derwent World Patents Index (DWPI)
- 高质量专利摘要和分类
- 引文分析功能

**优势**：

- 数据权威性高
- 分析工具专业

**不足**：

- 价格昂贵
- 学习曲线陡峭

#### 2.2.4 Orbit Intelligence（Questel）

**产品概述**：Orbit Intelligence是领先的专利分析和搜索软件。

**核心功能**：

- 专利景观分析
- 专利组合管理
- 先验技术搜索
- 竞争情报分析

**技术特点**：

- 全球专利数据覆盖
- 可视化分析工具
- 协作功能

### 2.4 差异化定位

基于竞品分析，本系统的差异化定位为：

1. **LLM原生架构**：以大语言模型为核心，而非传统检索+AI辅助
2. **实体和领域增强**：独特的实体识别和层次化领域知识增强机制
3. **双模式支持**：同时支持在线API和离线本地部署
4. **开源可定制**：面向学术研究和中小企业的开源解决方案

## 3. 用户需求分析

### 3.1 目标用户群体

#### 3.1.1 科研人员

**用户画像**：

- 高校、研究机构的研究人员
- 需要进行技术调研和专利分析
- 关注技术创新点和研究空白

**核心需求**：

- 快速了解某技术领域的专利布局
- 发现与自身研究相关的专利技术
- 评估研究成果的新颖性

#### 3.1.2 企业知识产权部门

**用户画像**：

- 企业IP专员、专利工程师
- 负责专利申请、维护和管理
- 需要进行FTO分析和竞争情报

**核心需求**：

- 专利侵权风险分析
- 竞争对手专利监控
- 专利价值评估

#### 3.1.3 技术转移机构

**用户画像**：

- 技术转移中心、孵化器工作人员
- 负责技术成果转化和对接
- 需要匹配技术供需双方

**核心需求**：

- 技术供需匹配
- 专利技术评估
- 潜在合作方发现

### 3.2 用户场景分析

#### 场景1：技术新颖性检索

**场景描述**：科研人员在撰写专利申请前，需要检索相关先验技术。

**用户流程**：

1. 输入技术方案描述
2. 系统自动提取关键实体和领域信息
3. 基于LLM进行语义匹配
4. 返回相关专利列表和相似度评分
5. 查看详细对比分析

#### 场景2：竞争对手专利监控

**场景描述**：企业IP部门需要监控竞争对手的专利动态。

**用户流程**：

1. 设置监控关键词和竞争对手
2. 系统定期抓取新专利
3. 自动进行实体识别和分类
4. 推送相关专利预警
5. 生成分析报告

#### 场景3：技术供需匹配

**场景描述**：技术转移机构需要为技术需求方匹配合适的专利技术。

**用户流程**：

1. 录入技术需求描述
2. 系统解析需求中的实体和领域
3. 在专利库中进行语义匹配
4. 返回匹配的专利技术列表
5. 提供技术对接建议

| F3.6     | 相似度评分   | 提供多维度的相似度评分          | P1     |

**技术匹配算法设计**：

```
匹配流程：
1. 输入查询文本
   │
   ▼
2. 实体和领域提取
   │
   ▼
3. 构建增强查询向量
   │
   ├──▶ 文本语义向量（LLM Embedding）
   │
   ├──▶ 实体特征向量
   │
   └──▶ 领域特征向量
   │
   ▼
4. 候选专利召回
   │
   ▼
5. LLM精排（考虑实体和领域匹配度）
   │
   ▼
6. 返回匹配结果
```

**相似度计算公式**：

$$
Similarity = \alpha \cdot Sim_{semantic} + \beta \cdot Sim_{entity} + \gamma \cdot Sim_{domain}
$$

其中：

- $Sim_{semantic}$：语义相似度（基于LLM Embedding）
- $Sim_{entity}$：实体匹配度（Jaccard相似度）
- $Sim_{domain}$：领域匹配度（IPC分类距离）
- $\alpha, \beta, \gamma$：权重参数（可配置）
