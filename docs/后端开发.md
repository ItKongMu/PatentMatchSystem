## 1. 项目概述

本项目是《基于大语言模型进行实体和领域增强的专利技术匹配系统》的后端服务，采用Spring Boot 3.2 + Spring AI 1.0.3技术栈，实现专利PDF上传、LLM实体提取、向量化存储、语义匹配等核心功能。

### 1.1 核心价值

- **LLM实体提取**：利用大语言模型从专利文本中自动识别技术实体（产品、方法、材料等）
- **领域层次分类**：自动推断IPC分类层次结构
- **语义向量匹配**：基于Qdrant向量数据库的高效相似度检索
- **混合检索策略**：结合向量检索和关键词检索，提升匹配精度

## 4. 核心功能

### 4.1 专利上传与解析

```
用户上传PDF → MinIO存储 → Spring AI PDF解析 → MySQL保存基础信息
                                    │
                                    ▼
                    ┌───────────────────────────────┐
                    │     LLM实体+领域提取          │
                    │   (通义千问/Ollama Qwen2.5)   │
                    └───────────────────────────────┘
                           │              │
                           ▼              ▼
                    ┌───────────┐  ┌───────────┐
                    │  实体列表  │  │  IPC分类  │
                    └───────────┘  └───────────┘
                                  │
                                  ▼
                    ┌───────────────────────────────┐
                    │      向量化 (Embedding)       │
                    │    通义千问 text-embedding-v3  │
                    └───────────────────────────────┘
                                  │
                                  ▼
                           ┌───────────┐
                           │  Qdrant   │
                           │ 向量+元数据│
                           └───────────┘
```

### 4.2 技术匹配流程

```
查询文本 → LLM提取实体/领域 → 向量检索(Qdrant) → LLM精排 → 返回匹配结果
```

### 4.3 实体类型

| 类型 | 说明 | 示例 |
|------|------|------|
| PRODUCT | 产品/设备/装置 | 深度学习处理器、图像识别模块 |
| METHOD | 方法/工艺/流程 | 卷积神经网络算法、特征提取方法 |
| MATERIAL | 材料/物质 | 碳纤维复合材料、纳米颗粒 |
| COMPONENT | 组件/部件 | 传感器阵列、控制单元 |
| EFFECT | 效果/性能 | 高精度识别、低功耗运行 |
| APPLICATION | 应用场景/用途 | 医疗图像诊断、自动驾驶 |

### 4.4 领域层级（IPC分类）

| 层级 | 说明 | 示例 |
|------|------|------|
| 1 | 部(Section) | G-物理 |
| 2 | 大类(Class) | G06-计算；计数 |
| 3 | 小类(Subclass) | G06F-电数字数据处理 |
| 4 | 主组(Main Group) | G06F16-信息检索 |
| 5 | 分组(Group) | G06F16/30-代码转换 |

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `DASHSCOPE_API_KEY` | 通义千问API密钥 | - |
| `LLM_MODE` | LLM模式(online/offline) | online |
| `MYSQL_HOST` | MySQL主机 | localhost |
| `MYSQL_PORT` | MySQL端口 | 3306 |
| `MYSQL_PASSWORD` | MySQL密码 | root |
| `REDIS_HOST` | Redis主机 | localhost |
| `ES_URIS` | Elasticsearch地址 | http://localhost:9200 |
| `QDRANT_HOST` | Qdrant主机 | localhost |
| `MINIO_ENDPOINT` | MinIO端点 | http://localhost:9000 |

### 6.2 LLM双模式配置

**在线模式（通义千问）**：
```yaml
spring:
  ai:
    dashscope:
      api-key: ${DASHSCOPE_API_KEY}
      chat:
        options:
          model: qwen-plus
      embedding:
        options:
          model: text-embedding-v3

patent:
  llm-mode: online
```

**离线模式（Ollama）**：
```yaml
spring:
  ai:
    ollama:
      base-url: http://localhost:11434
      chat:
        options:
          model: qwen2.5:7b
      embedding:
        options:
          model: nomic-embed-text

patent:
  llm-mode: offline
```

## 7. 快速启动

### 7.1 环境要求

- JDK 17+
- Maven 3.8+
- Docker & Docker Compose（可选）

### 7.2 本地开发启动

**1. 启动基础设施（Docker Compose）**

```bash
# 进入项目根目录
cd PatentMatchSystem

# 仅启动基础设施服务
docker-compose up -d mysql redis elasticsearch qdrant minio
```

**2. 初始化数据库**

```bash
# 连接MySQL执行初始化脚本
mysql -h localhost -u root -p < backend/src/main/resources/db/init.sql
```

**3. 配置环境变量**

```bash
# 复制环境变量示例
cp .env.example .env

# 编辑.env文件，填入通义千问API密钥
# DASHSCOPE_API_KEY=your-api-key
```

**4. 启动后端服务**

```bash
cd backend

# 方式1：Maven启动
mvn spring-boot:run

# 方式2：先打包再启动
mvn clean package -DskipTests
java -jar target/patent-match-system-1.0.0-SNAPSHOT.jar
```

**5. 验证启动**

- API文档：http://localhost:8080/doc.html
- 健康检查：http://localhost:8080/actuator/health

### 7.3 Docker一键部署

```bash
# 进入项目根目录
cd PatentMatchSystem

# 配置环境变量
cp .env.example .env
# 编辑.env，填入DASHSCOPE_API_KEY

# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f backend
```

### 7.4 离线模式部署

```bash
# 启动包含Ollama的服务
docker-compose --profile offline up -d

# 拉取本地模型
docker exec patent-ollama ollama pull qwen2.5:7b
docker exec patent-ollama ollama pull nomic-embed-text

# 设置LLM_MODE=offline
export LLM_MODE=offline
```

### C. 常见问题

**Q: 启动报错找不到DashScope API Key？**  
A: 设置环境变量 `DASHSCOPE_API_KEY`，或在 `application.yml` 中配置。

**Q: PDF解析中文乱码？**  
A: 确保PDF文件使用UTF-8编码，或调整Spring AI PDF Reader配置。

**Q: 向量检索结果不准确？**  
A: 检查Qdrant collection是否正确初始化，调整相似度阈值。
