## 二、联调问题排查记录

### 2.1 用户注册失败问题

#### 错误现象

```
2026-02-02 16:35:40 ERROR - Exception during pool initialization.
java.sql.SQLSyntaxErrorException: Unknown database 'ppatent_match_system'
```

#### 问题原因

**数据库名配置错误**：数据库名多了一个字母 `p`，应为 `patent_match_system`，实际连接的是 `ppatent_match_system`。

#### 解决方案

1. **检查 application.yml 配置**
   确认数据源 URL 配置正确：

   ```yaml
   spring:
     datasource:
       url: jdbc:mysql://${MYSQL_HOST:localhost}:${MYSQL_PORT:3306}/patent_match_system?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
   ```
2. **检查环境变量**
   如果使用了环境变量，请确认没有设置错误的数据库名：

   ```bash
   # Windows PowerShell 查看环境变量
   $env:MYSQL_HOST

   # 或检查 .env 文件
   ```
3. **创建数据库**
   如果数据库不存在，请先创建：

   ```sql
   CREATE DATABASE IF NOT EXISTS patent_match_system 
   DEFAULT CHARACTER SET utf8mb4 
   DEFAULT COLLATE utf8mb4_unicode_ci;
   ```
4. **执行数据库初始化脚本**
   确保已执行建表语句，包括 `sys_user` 表：

   ```sql
   CREATE TABLE IF NOT EXISTS sys_user (
       id BIGINT AUTO_INCREMENT PRIMARY KEY,
       username VARCHAR(50) NOT NULL UNIQUE COMMENT '用户名',
       password VARCHAR(100) NOT NULL COMMENT '密码',
       nickname VARCHAR(50) COMMENT '昵称',
       avatar VARCHAR(255) COMMENT '头像',
       status TINYINT DEFAULT 1 COMMENT '状态：0-禁用，1-正常',
       deleted TINYINT DEFAULT 0 COMMENT '删除标记',
       create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
       update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
   ) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统用户表';
   ```

### 2.2 favicon.ico 404 问题

#### 错误现象

```
2026-02-02 16:33:39 ERROR - NoResourceFoundException: No static resource favicon.ico.
```

#### 问题原因

浏览器访问后端接口时自动请求 `/favicon.ico`，但后端未配置静态资源处理。

#### 解决方案

**方案一：在全局异常处理器中忽略此错误**

修改 `GlobalExceptionHandler.java`：

```java
package com.patent.common.exception;

import cn.dev33.satoken.exception.NotLoginException;
import cn.dev33.satoken.exception.NotPermissionException;
import com.patent.common.Result;
import lombok.extern.slf4j.Slf4j;
import org.springframework.http.HttpStatus;
import org.springframework.validation.BindException;
import org.springframework.validation.FieldError;
import org.springframework.web.bind.MethodArgumentNotValidException;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.ResponseStatus;
import org.springframework.web.bind.annotation.RestControllerAdvice;
import org.springframework.web.servlet.resource.NoResourceFoundException;

import java.util.List;
import java.util.stream.Collectors;

@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    // ... 其他异常处理方法 ...

    /**
     * 处理静态资源未找到异常（如 favicon.ico）
     */
    @ExceptionHandler(NoResourceFoundException.class)
    @ResponseStatus(HttpStatus.NOT_FOUND)
    public Result<Void> handleNoResourceFoundException(NoResourceFoundException e) {
        // 对于 favicon.ico 不记录错误日志
        if (!e.getResourcePath().contains("favicon")) {
            log.warn("资源未找到: {}", e.getResourcePath());
        }
        return Result.error(404, "资源不存在");
    }

    /**
     * 处理其他异常
     */
    @ExceptionHandler(Exception.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public Result<Void> handleException(Exception e) {
        log.error("系统异常", e);
        return Result.error("系统异常，请稍后重试");
    }
}
```

**方案二：添加空的 favicon.ico 处理**

在 `resources/static/` 目录下放置一个 favicon.ico 文件，或添加一个 Controller：

```java
@RestController
public class FaviconController {
  
    @GetMapping("favicon.ico")
    @ResponseStatus(HttpStatus.NO_CONTENT)
    public void favicon() {
        // 返回空响应
    }
}
```

## 六、常见问题 FAQ

### Q1: 注册时提示"系统异常，请稍后重试"

**可能原因：**

1. 数据库连接失败（检查数据库名是否正确）
2. 数据库表不存在
3. Redis 连接失败

**排查步骤：**

1. 查看后端控制台日志
2. 检查 `application.yml` 配置
3. 验证 MySQL、Redis 服务状态

### Q2: 前端请求 404

**可能原因：**

1. Vite 代理未正确配置
2. 后端 API 路径不匹配

**排查步骤：**

1. 确认 `vite.config.js` 代理配置
2. 对比前端请求 URL 与后端 Controller 路径
3. 使用 Swagger UI 直接测试后端接口：http://localhost:8080/doc.html

### Q3: 登录后获取用户信息 401

**可能原因：**

1. Token 未正确传递
2. Token 已过期
3. Redis session 丢失

**排查步骤：**

1. 检查请求头是否包含 Authorization
2. 检查 token 格式是否正确
3. 验证 Redis 中 session 数据

## 九、专利处理流程

### 9.1 处理流程图

```
┌─────────────┐
│  上传专利   │
│ (PDF/文本)  │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   PENDING   │ ← 初始状态
└──────┬──────┘
       │ 触发处理
       ▼
┌─────────────┐
│   PARSING   │ ← PDF解析（仅PDF上传）
│  解析PDF    │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ EXTRACTING  │ ← LLM实体提取
│ 提取实体    │   调用通义千问/Ollama
│ 识别领域    │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ VECTORIZING │ ← 向量化存储
│ 生成向量    │   存入Qdrant
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   SUCCESS   │ ← 处理完成
└─────────────┘

异常情况 → FAILED（记录错误信息）
```

### 9.2 处理流程说明

| 阶段 | 状态 | 说明 |
|------|------|------|
| 1. 上传 | PENDING | 专利上传后初始状态 |
| 2. 解析 | PARSING | 解析PDF文件内容（仅PDF上传） |
| 3. 提取 | EXTRACTING | 调用LLM提取技术实体和领域信息 |
| 4. 向量化 | VECTORIZING | 生成文本向量并存入Qdrant |
| 5. 完成 | SUCCESS | 处理完成，可进行匹配 |
| 异常 | FAILED | 处理失败，记录错误原因 |

### 9.3 LLM配置

系统支持两种LLM模式：

**在线模式 (online) - 通义千问：**
```yaml
patent:
  llm-mode: online

spring:
  ai:
    openai:
      base-url: https://dashscope.aliyuncs.com/compatible-mode
      api-key: ${DASH_SCOPE_API_KEY}
      connect-timeout: 60s
      read-timeout: 180s
```

**离线模式 (offline) - Ollama：**
```yaml
patent:
  llm-mode: offline

spring:
  ai:
    ollama:
      base-url: http://localhost:11434
```

### 9.4 常见问题

#### Q1: LLM调用超时
**错误信息：** `timeout` 或 `I/O error on POST request`

**解决方案：**
1. 检查网络连接是否正常
2. 确认 `DASH_SCOPE_API_KEY` 环境变量已正确设置
3. 增加超时时间配置（已默认设置为180秒）
4. 考虑切换到离线模式使用Ollama

#### Q2: 实体提取结果为空
**可能原因：**
1. LLM调用失败（超时或API错误）
2. 输入文本过短或格式不正确
3. API Key无效或额度不足

**排查步骤：**
1. 查看后端日志中的 `LLM实体提取` 相关输出
2. 检查通义千问控制台的API调用记录
3. 尝试切换到Ollama离线模式测试

#### Q3: 向量存储失败
**可能原因：**
1. Qdrant服务未启动
2. Qdrant连接配置错误
3. 向量维度不匹配

**解决方案：**
1. 确认Qdrant服务运行正常：`curl http://192.168.65.131:6333/health`
2. 检查 `application.yml` 中的Qdrant配置
3. 确保向量维度配置正确（在线模式1024，离线模式768）
