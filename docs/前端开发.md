## 1. 项目概述

本项目是专利技术匹配系统的前端部分，基于 Vue 3 + Element Plus 构建，提供专利上传、管理、检索和智能匹配等功能的用户界面。

### 1.1 核心功能

| 功能模块 | 描述 |
|---------|------|
| 用户认证 | 登录、注册、退出登录 |
| 专利上传 | PDF文件上传、文本录入 |
| 专利管理 | 专利列表、详情、删除、触发处理 |
| 技术匹配 | 文本查询匹配、专利相似匹配 |
| 匹配历史 | 查看历史匹配记录 |
| 专利检索 | 快速检索、高级检索（ES全文检索） |

## 4. 功能模块

### 4.1 用户认证

**登录页面 (`/login`)**
- 用户名/密码登录
- 表单验证
- 登录成功后跳转

**注册页面 (`/register`)**
- 用户名、密码、确认密码
- 密码一致性校验
- 注册成功跳转登录

### 4.2 专利上传

**上传页面 (`/patent/upload`)**

| 上传方式 | 说明 |
|---------|------|
| PDF上传 | 拖拽或选择PDF文件上传，支持最大50MB |
| 文本录入 | 手动输入专利标题、摘要等信息 |

**上传流程**：
1. 选择上传方式（PDF/文本）
2. 填写专利信息
3. 提交上传
4. 可选择立即触发处理流程

### 4.3 专利列表

**列表页面 (`/patent/list`)**

| 功能 | 说明 |
|------|------|
| 搜索 | 按关键词、解析状态筛选 |
| 分页 | 支持切换每页数量 |
| 操作 | 查看详情、触发处理、相似匹配、删除 |

**解析状态**：
- `PENDING` - 待处理
- `PARSING` - PDF解析中
- `EXTRACTING` - LLM实体提取中
- `VECTORIZING` - 向量化存储中
- `SUCCESS` - 处理完成
- `FAILED` - 处理失败

### 4.4 专利详情

**详情页面 (`/patent/detail/:id`)**

展示内容：
- 基本信息（标题、公开号、申请人等）
- 专利摘要
- 技术实体（LLM提取）
- 技术领域（IPC分类层次）
- 向量信息

### 4.5 技术匹配

**匹配页面 (`/match`)**

| 匹配模式 | 说明 |
|---------|------|
| 文本查询 | 输入技术描述，通过LLM分析后进行语义匹配 |
| 专利匹配 | 选择已处理专利，查找相似专利 |

**匹配结果展示**：
- 查询实体分析
- 匹配专利列表（排序按相似度）
- 相似度分数
- 匹配原因说明
- 实体匹配数量
- 领域是否匹配

### 4.6 专利检索

**检索页面 (`/search`)**

| 检索方式 | 说明 |
|---------|------|
| 快速检索 | 输入关键词，全文检索标题、摘要、实体 |
| 高级检索 | 多条件组合检索（标题、摘要、申请人、IPC分类、实体类型） |

## 5. API接口

### 5.1 接口对应表

前端API与后端RESTful接口完全对应：

**认证接口 (`/api/auth`)**

| 前端方法 | 后端接口 | 说明 |
|---------|---------|------|
| `authApi.login()` | `POST /api/auth/login` | 用户登录 |
| `authApi.register()` | `POST /api/auth/register` | 用户注册 |
| `authApi.logout()` | `POST /api/auth/logout` | 退出登录 |
| `authApi.getUserInfo()` | `GET /api/auth/info` | 获取用户信息 |

**专利接口 (`/api/patent`)**

| 前端方法 | 后端接口 | 说明 |
|---------|---------|------|
| `patentApi.upload()` | `POST /api/patent/upload` | PDF上传 |
| `patentApi.createByText()` | `POST /api/patent/text` | 文本录入 |
| `patentApi.process()` | `POST /api/patent/process/{id}` | 触发处理 |
| `patentApi.getDetail()` | `GET /api/patent/{id}` | 获取详情 |
| `patentApi.getEntities()` | `GET /api/patent/{id}/entities` | 获取实体 |
| `patentApi.getDomains()` | `GET /api/patent/{id}/domains` | 获取领域 |
| `patentApi.getVector()` | `GET /api/patent/{id}/vector` | 获取向量 |
| `patentApi.getList()` | `GET /api/patent/list` | 获取列表 |
| `patentApi.delete()` | `DELETE /api/patent/{id}` | 删除专利 |

**匹配接口 (`/api/match`)**

| 前端方法 | 后端接口 | 说明 |
|---------|---------|------|
| `matchApi.match()` | `POST /api/match` | 文本匹配 |
| `matchApi.matchByPatent()` | `POST /api/match/patent/{id}` | 专利匹配 |
| `matchApi.getHistory()` | `GET /api/match/history` | 匹配历史 |

**检索接口 (`/api/search`)**

| 前端方法 | 后端接口 | 说明 |
|---------|---------|------|
| `searchApi.search()` | `GET /api/search` | 快速检索 |
| `searchApi.advancedSearch()` | `POST /api/search/advanced` | 高级检索 |

## 6. 快速开始

### 6.1 环境要求

- Node.js 18.x 或 20.x LTS
- npm 或 pnpm 包管理器

### 6.2 安装依赖

```bash
cd frontend

# 使用 npm
npm install

# 或使用 pnpm（推荐）
pnpm install
```

### 6.3 开发模式启动

```bash
# 启动开发服务器
npm run dev

# 或
pnpm dev
```

启动后访问：http://localhost:5173

### 6.4 开发环境代理配置

开发模式下，`/api` 请求会自动代理到 `http://localhost:8080`，配置在 `vite.config.js`：

```javascript
server: {
  port: 5173,
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true
    }
  }
}
```

### 6.5 构建生产版本

```bash
# 构建
npm run build

# 预览构建结果
npm run preview
```

构建产物输出到 `dist/` 目录。

## 7. 开发指南

### 7.1 添加新页面

1. 在 `src/views/` 下创建页面组件
2. 在 `src/router/index.js` 添加路由配置
3. 如需侧边栏菜单，在 `MainLayout.vue` 添加菜单项

### 7.2 添加新API

1. 在 `src/api/` 下创建或修改对应模块
2. 在 `src/api/index.js` 导出

示例：
```javascript
// src/api/example.js
import request from './request'

export const exampleApi = {
  getData(params) {
    return request({
      url: '/example/data',
      method: 'get',
      params
    })
  }
}
```

### 7.3 状态管理

使用 Pinia 进行状态管理，Store 定义在 `src/stores/` 下：

```javascript
// src/stores/example.js
import { defineStore } from 'pinia'
import { ref } from 'vue'

export const useExampleStore = defineStore('example', () => {
  const data = ref([])
  
  async function fetchData() {
    // ...
  }
  
  return { data, fetchData }
})
```

### 7.4 样式规范

- 使用 SCSS 编写样式
- 组件样式使用 `scoped`
- 全局样式在 `src/styles/index.scss`
- 遵循 BEM 命名规范

### 7.5 Element Plus 组件

已配置自动导入，直接在模板中使用即可：

```vue
<template>
  <el-button type="primary">按钮</el-button>
  <el-table :data="tableData">...</el-table>
</template>
```

## 8. 生产部署

### 8.1 Nginx 配置

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    root /path/to/dist;
    index index.html;
    
    # 静态资源缓存
    location /assets {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # API代理
    location /api {
        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    
    # SPA路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### 8.2 Docker 部署

**Dockerfile**：
```dockerfile
# 构建阶段
FROM node:20-alpine as builder

WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

# 运行阶段
FROM nginx:alpine

COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 8.3 环境变量

生产环境配置在 `.env.production`：

```bash
VITE_APP_TITLE=专利技术匹配系统
VITE_API_BASE_URL=/api
```
