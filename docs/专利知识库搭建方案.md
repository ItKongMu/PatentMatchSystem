```
┌─────────────────────────────────────────────────────────────────────┐
│                    专利技术匹配系统 (PatentMatchSystem)              │
└────────────────────────────┬────────────────────────────────────────┘
                             │
        ┌────────────────────┼────────────────────┬──────────────────┐
        ▼                    ▼                    ▼                  ▼
┌──────────────┐    ┌──────────────────┐   ┌────────────────┐  ┌──────────────┐
│ 结构化数据仓储│    │ 知识图谱层 (Neo4j)│   │ 向量语义层(Qdrant)│  │ 全文检索(ES) │
│ (MySQL)      │    │ 实体/关系/事件    │   │ 概念/专利/实体向量 │  │ 标题摘要索引 │
└──────────────┘    └──────────────────┘   └────────────────┘  └──────────────┘
        │                    │                    │                  │
        ▼                    ▼                    ▼                  ▼
┌──────────────┐    ┌──────────────────┐   ┌────────────────┐  ┌──────────────┐

│ 领域词典数据  │    │ 实体别名/同义词 │   │ 图谱实体向量   │  │ 实体文本索引  │
└──────────────┘    └──────────────────┘   └────────────────┘  └──────────────┘
```

### 1.2 核心能力替换关系

| 能力     | 原知识库形态    | 知识图谱升级后                   | 价值提升           |
| -------- | --------------- | -------------------------------- | ------------------ |
| 领域层次 | IPC分类表       | `IPC`节点 + `PARENT_OF`层次  | 可图查询、路径推理 |

| 领域知识 | 描述文本        | `Concept`节点 + `RELATED_TO` | 结构化语义增强     |

| 节点类型        | 说明         | 示例                 |
| --------------- | ------------ | -------------------- |
| `Patent`      | 专利文献     | CN201910123456A      |
| `IPC`         | IPC/CPC分类  | G06F16/30            |
| `Entity`      | 专利技术实体 | “图像传感器”       |

| `Component`   | 组件部件     | “电源模块”         |

| `Concept`     | 领域概念     | “图像识别”         |

### 2.2 关系类型设计

| 关系类型          | 起点 -> 终点              | 说明         |
| ----------------- | ------------------------- | ------------ |
| `HAS_IPC`       | Patent -> IPC             | 专利所属IPC  |
| `MENTIONS`      | Patent -> Entity          | 专利包含实体 |
| `USES_METHOD`   | Patent -> Method          | 专利采用方法 |
| `HAS_COMPONENT` | Patent -> Component       | 专利组件     |
| `USES_MATERIAL` | Patent -> Material        | 专利材料     |
| `APPLIED_IN`    | Patent -> Application     | 应用场景     |
| `FILED_BY`      | Patent -> Applicant       | 申请人       |
| `RELATED_TO`    | Entity/Concept -> Concept | 语义关联     |
| `SAME_AS`       | Entity -> Entity          | 同义词/别名  |
| `PARENT_OF`     | IPC -> IPC                | 分类层次     |
| `CITES`         | Patent -> Patent          | 引用关系     |

### 2.3 关系属性示例

| 关系           | 属性           | 说明           |
| -------------- | -------------- | -------------- |
| `MENTIONS`   | `confidence` | 实体抽取置信度 |
| `RELATED_TO` | `weight`     | 关联强度       |
| `CITES`      | `source`     | 引证来源       |

## 3. 图谱数据源与抽取策略

### 3.1 可复用的原知识库数据源

| 原方案模块 | 可复用项         | 图谱化映射                       |
| ---------- | ---------------- | -------------------------------- |
| IPC分类库  | WIPO IPC数据     | `IPC`节点 + `PARENT_OF`关系  |
| 实体术语库 | 实体词典/同义词  | `Entity`节点 + `SAME_AS`关系 |
| 专利语料   | 专利文本         | `Patent`节点 + 关系抽取        |
| 领域描述   | IPC描述/行业词典 | `Concept`节点 + `RELATED_TO` |

### 3.2 LLM+规则抽取策略（Superpower Brainstorm）

**Brainstorm输出：抽取与融合流程**

1. **领域抽取**：从专利标题、摘要、权利要求中抽取 IPC/领域概念
2. **实体抽取**：识别产品、方法、组件、材料、效果、应用场景
3. **关系抽取**：通过依存语法/Prompt模板抽取“使用/包含/作用于”等关系
4. **实体对齐**：与既有术语库进行同义词归并、别名聚合
5. **置信过滤**：低置信度候选进入“待确认”节点或人工复核队列
6. **图谱写入**：统一ID生成策略 + 幂等写入

## 4. 图谱构建与写入流程

### 4.1 总体流程（推荐规范）

```
数据采集 -> 预处理清洗 -> LLM抽取实体/关系 -> 实体对齐/消歧 -> 图谱写入 -> 向量化补充 -> 评估与更新
```

### 4.2 Graph ETL流水线

1. **采集阶段**：

   - PDF/文本/CSV 解析
   - 申请人标准化
   - IPC编码清洗
2. **抽取阶段**：

   - LLM+规则混合抽取实体与关系
   - 生成 `GraphCandidate`
3. **对齐阶段**：

   - 使用实体词库匹配（精确/模糊/向量）
   - 实体统一ID与别名归并
4. **图谱写入**：

   - **初次导入：** `neo4j-admin database import full`（CSV 节点/关系文件）
   - **增量更新：** Spring Data Neo4j API 写入（PDF/文本/用户输入）
   - **CSV 批量更新：** 普通用户 CSV 默认增量写入；管理员可触发全量重建
5. **索引与向量化**：

   - 图谱实体向量入Qdrant
   - 专利全文入ES

## 5. 图谱与RAG混合检索

### 5.1 三级检索融合

| 层级       | 方式         | 作用                 |
| ---------- | ------------ | -------------------- |
| 图检索     | Neo4j 图遍历 | 结构化推理、路径发现 |
| 向量检索   | Qdrant       | 语义召回             |
| 关键词检索 | ES           | 高精度匹配           |

### 5.2 图谱增强RAG

1. 用户问题 -> LLM解析实体/领域
2. 图谱中检索相关节点（实体、方法、IPC）
3. 将图谱邻接节点作为增强上下文
4. 再调用向量召回/ES过滤
5. 构建最终RAG上下文

### 5.3 专利匹配算法现状与图谱增强改进

**现有算法（已在项目中使用）：**

- 语义相似度：Embedding 余弦相似度（Qdrant）
- 实体匹配度：Jaccard 相似度（实体集合交并比）
- 领域匹配度：IPC 分类距离
- 综合评分：加权融合

**引入知识图谱后的改进方向：**

1. **图谱路径增强匹配**
   - 利用 `Patent -> Entity -> Patent` 的二跳路径进行关联扩展
   - 结合 `IPC` 层次路径相似度做结构化加分
2. **关系权重加权**
   - 对 `MENTIONS/USES_METHOD/HAS_COMPONENT` 等关系引入权重，替代纯集合 Jaccard
3. **图谱邻接召回融合**
   - 以图谱邻接节点召回候选专利，再与向量/ES 融合排序
4. **图谱一致性校验**
   - 若实体/IPC 在图谱中不存在或异常，可回退到原算法

**图谱增强匹配评分公式（建议）**

设：
- \(S_{vec}\)：语义向量相似度（余弦）
- \(S_{ent}\)：实体匹配度（加权 Jaccard）
- \(S_{ipc}\)：IPC 层次相似度（路径距离归一化）
- \(S_{graph}\)：图谱路径增强分（基于二跳/三跳关联）

综合评分：

\[
Score = w_1 S_{vec} + w_2 S_{ent} + w_3 S_{ipc} + w_4 S_{graph}
\]

其中：
- \(w_1 + w_2 + w_3 + w_4 = 1\)
- 默认建议：\(w_1=0.5, w_2=0.2, w_3=0.2, w_4=0.1\)

图谱路径分示例：

\[
S_{graph} = \alpha \cdot \frac{N_{2hop}}{N_{2hop}+1} + (1-\alpha) \cdot \frac{N_{3hop}}{N_{3hop}+1}
\]

- \(N_{2hop}\)：二跳可达的关联专利数量
- \(N_{3hop}\)：三跳可达的关联专利数量
- \(\alpha\) 建议 0.7（更偏重二跳关系）

### 6.1 质量指标（沿用并调整）

## 8. 前端图谱展示接口建议

### 8.1 接口建议（后端提供）

- `GET /api/graph/patent/{publicationNo}`：返回某专利的图谱邻接节点与关系（用于专利详情页可视化）
- `GET /api/graph/entity/{entityName}`：返回某实体的关联专利/方法/组件（用于实体关系图）
- `GET /api/graph/ipc/{ipcCode}`：返回 IPC 层次结构与关联专利数（用于领域可视化）
- `POST /api/graph/query`：通用图谱查询（支持节点类型、关系类型、深度、过滤条件）
- `GET /api/graph/csv-template`：下载图谱 CSV 模板（用于管理员全量导入）

### 8.2 返回数据结构建议（ECharts/Graph 通用）

```json
{
  "nodes": [
    {"id": "Patent:CN201910123456A", "label": "Patent", "name": "一种图像识别方法"},
    {"id": "Entity:图像传感器", "label": "Entity", "name": "图像传感器"}
  ],
  "links": [
    {"source": "Patent:CN201910123456A", "target": "Entity:图像传感器", "type": "MENTIONS"}
  ]
}
```

> 说明：该结构可直接用于 ECharts graph 或前端自定义关系图组件。

**专利-IPC（has_ipc_rels.csv）**

**专利-实体（mentions_rels.csv）**

```

```


### 9.3 CSV 导入示例

```bash
neo4j-admin database import full \
  --nodes=Patent=patent_nodes.csv \
  --nodes=IPC=ipc_nodes.csv \
  --nodes=Entity=entity_nodes.csv \
  --nodes=Applicant=applicant_nodes.csv \
  --relationships=HAS_IPC=has_ipc_rels.csv \
  --relationships=MENTIONS=mentions_rels.csv \
  --relationships=FILED_BY=filed_by_rels.csv \
  --relationships=PARENT_OF=ipc_parent_rels.csv \
  neo4j
```

## 10. 附录：关键配置与规范示例

### 8.1 Neo4j连接配置（Spring Boot 3.x）

```properties
spring.neo4j.uri=bolt://localhost:7687
spring.neo4j.authentication.username=neo4j
spring.neo4j.authentication.password=secret
```

### 8.2 图谱实体建模规范（Spring Data Neo4j）

```java
@Node
public class PatentNode {
    @Id @GeneratedValue
    private Long id;

    private String publicationNo;
    private String title;
    private String abstractText;
}

@Node
public class IpcNode {
    @Id
    private String ipcCode;

    private String titleCn;
    private Integer level;
}

@RelationshipProperties
public class MentionedEntityRel {
    @RelationshipId
    private Long id;

    @TargetNode
    private EntityNode entity;

    private Double confidence;
}
```

### 8.3 约束与索引初始化（Cypher）

```cypher
CREATE CONSTRAINT IF NOT EXISTS
FOR (p:Patent)
REQUIRE p.publicationNo IS NODE UNIQUE;

CREATE CONSTRAINT IF NOT EXISTS
FOR (i:IPC)
REQUIRE i.ipcCode IS NODE UNIQUE;

CREATE CONSTRAINT IF NOT EXISTS
FOR (e:Entity)
REQUIRE e.name IS NODE UNIQUE;

CREATE FULLTEXT INDEX IF NOT EXISTS patentFulltext
FOR (p:Patent)
ON EACH [p.title, p.abstractText];
```

### 8.4 初次导入（neo4j-admin）

```bash
neo4j-admin database import full \
  --nodes=Patent=patent_nodes.csv \
  --nodes=IPC=ipc_nodes.csv \
  --nodes=Entity=entity_nodes.csv \
  --relationships=MENTIONS=mentions_rels.csv \
  neo4j
```

### 8.5 增量写入（Spring Data Neo4j Service 伪代码）

```java
@Service
public class PatentGraphService {
    private final PatentRepository patentRepository;

    public PatentGraphService(PatentRepository patentRepository) {
        this.patentRepository = patentRepository;
    }

    @Transactional
    public void upsertPatent(PatentNode patent) {
        patentRepository.save(patent);
    }
}
```

## 结语

本方案在原“专利知识库搭建方案”基础上升级为**知识图谱构建方案**，充分复用原有数据源、实体体系与检索架构，在此之上引入图数据库与结构化关系表达能力。该方案支持：

- 结构化语义关联
- 关系推理与图遍历
- 与向量检索、全文检索的混合增强

为专利技术匹配与智能检索提供更高质量的语义支撑。
