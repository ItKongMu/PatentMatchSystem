## 1. 项目概述

### 1.1 项目简介

本系统是一个基于大语言模型（LLM）的智能专利技术匹配平台，通过 LLM 的语义理解能力，结合实体识别和层次化领域知识增强技术，实现专利文本的智能解析、检索和技术匹配。系统采用 B/S 架构，为用户提供专利录入、检索、技术匹配、智能对话、数据可视化等一站式服务。

### 1.2 核心特色

| 特色 | 说明 |
|------|------|
| **LLM 双模式** | 支持在线模式（通义千问 API）和离线模式（Ollama 本地部署） |
| **实体与领域增强** | 从专利文本中自动提取技术实体和 IPC 层次化领域信息，增强匹配精度 |
| **混合检索** | 向量语义检索（Qdrant）+ 全文关键词检索（Elasticsearch）相结合 |
| **智能对话** | 基于 Function Calling 的对话式专利检索，支持多轮对话和会话管理 |
| **数据可视化** | 实体词云、领域分布饼图、专利趋势折线图、申请人排行等统计展示 |

### 1.3 目标用户

- **科研人员**：技术调研、专利分析、新颖性检索
- **企业知识产权部门**：FTO 分析、竞争对手监控、专利价值评估
- **技术转移机构**：技术供需匹配、专利技术评估

### 2.2 核心数据处理流程

**专利处理流程：**

```
PDF上传/文本录入 → MinIO存储/直接入库 → PDF解析(Spring AI) → LLM实体/领域提取 
    → 向量化(Embedding) → Qdrant存储 + ES索引 → 处理完成
```

**技术匹配流程：**

```
查询文本/专利ID → LLM提取实体/领域 → 混合检索(Qdrant+ES) → LLM精排 
    → 相似度计算(语义+实体+领域) → 返回匹配结果
```

## 4. 功能模块详解

### 4.1 用户认证模块

| 功能 | 说明 |
|------|------|
| 用户注册 | 用户名、密码、昵称注册，密码 BCrypt 加密 |
| 用户登录 | 用户名密码登录，返回 Sa-Token |
| 退出登录 | 清除登录态 |
| 获取用户信息 | 获取当前登录用户的昵称、角色等信息 |
| 路由守卫 | 未登录用户访问需认证页面时跳转登录 |

### 4.2 专利管理模块

| 功能 | 说明 |
|------|------|
| **PDF 上传** | 上传专利 PDF 文件，存储至 MinIO，创建专利记录，状态为 PENDING |
| **文本录入** | 直接输入专利标题、摘要、权利要求等文本创建专利 |
| **CSV 批量导入** | 支持 CSV 文件批量导入专利元数据，可预览后确认导入，可选自动触发处理 |
| **触发处理** | 对 PENDING 状态的专利触发解析流程：PDF 解析 → 实体/领域提取 → 向量化 |
| **专利列表** | 分页查询，支持按解析状态、关键词筛选 |
| **专利详情** | 查看专利基本信息、实体列表、领域层次、向量信息 |
| **删除专利** | 删除专利及其关联的实体、领域、向量、ES 索引、MinIO 文件 |

**解析状态流转：**

```
PENDING → PARSING → EXTRACTING → VECTORIZING → SUCCESS
    └──────────────────────────────────────────────→ FAILED
```

### 4.3 实体与领域提取模块（LLM 驱动）

| 功能 | 说明 |
|------|------|
| **实体识别** | 从专利文本中提取 7 类技术实体 |
| **领域分类** | 按 IPC 分类体系提取层次化领域信息 |

**实体类型：**

| 类型 | 说明 | 示例 |
|------|------|------|
| PRODUCT | 产品/装置 | 智能手机、传感器 |
| METHOD | 方法/工艺 | 深度学习算法、制造工艺 |
| MATERIAL | 材料 | 碳纤维、石墨烯 |
| COMPONENT | 组件/部件 | 处理器、电池 |
| EFFECT | 技术效果 | 提高效率、降低成本 |
| APPLICATION | 应用场景 | 医疗诊断、自动驾驶 |

**IPC 领域层级：**

| 层级 | 说明 | 示例 |
|------|------|------|
| 1 | 部(Section) | G-物理 |
| 2 | 大类(Class) | G06-计算 |
| 3 | 小类(Subclass) | G06F-电数字数据处理 |
| 4 | 主组(Main Group) | G06F16-信息检索 |
| 5 | 分组(Group) | G06F16/30-代码转换 |

### 4.4 技术匹配模块

| 功能 | 说明 |
|------|------|
| **文本匹配** | 输入自然语言描述，系统提取实体/领域后进行向量+关键词混合检索，LLM 精排后返回匹配专利 |
| **相似专利匹配** | 选择某一专利，基于其向量和元数据检索语义相似的专利 |
| **匹配历史** | 分页查看历史匹配记录，支持按匹配模式（PATENT/TEXT）筛选 |

**匹配算法：**

- 语义相似度：基于 Embedding 向量余弦相似度
- 实体匹配度：Jaccard 相似度
- 领域匹配度：IPC 分类距离
- 综合评分：加权融合上述指标

### 4.5 专利检索模块

| 功能 | 说明 |
|------|------|
| **快速检索** | 多字段匹配、短语提升、高亮显示，适用于首页搜索 |
| **关键词检索** | 兼容旧接口的简单关键词检索 |
| **高级检索** | 多条件组合：标题、摘要、申请人、日期范围、IPC 领域、实体类型等 |
| **深度分页** | 使用 search_after 实现，避免 from+size 超限 |
| **领域分布统计** | 按 IPC 部统计专利数量 |
| **申请人排行** | 统计专利数量最多的申请人 |
| **ES 健康检查** | 检查 Elasticsearch 可用性 |
| **初始化索引** | 创建 ES 索引和映射（幂等） |

### 4.6 智能对话模块

| 功能 | 说明 |
|------|------|
| **同步对话** | 发送消息后一次性返回 AI 回复 |
| **流式对话** | SSE 流式返回，实时显示生成内容 |
| **Function Calling** | AI 可调用专利检索、技术匹配等工具函数 |
| **会话管理** | 多会话支持，每个会话独立上下文 |
| **会话列表** | 获取当前用户的会话列表，分页 |
| **会话消息** | 获取指定会话的历史消息 |
| **删除会话** | 删除会话及其所有消息 |
| **更新会话标题** | 自定义会话标题 |
| **归档/恢复** | 归档不再使用的会话，支持恢复 |
| **建议问题** | 获取推荐的对话起始问题 |
| **MongoDB 持久化** | 聊天历史存储于 MongoDB，重启不丢失 |

### 4.7 专利收藏模块

| 功能 | 说明 |
|------|------|
| **添加收藏** | 将专利加入收藏夹，可设置备注和分组 |
| **取消收藏** | 从收藏夹移除 |
| **检查收藏状态** | 单个或批量检查专利是否已收藏 |
| **收藏列表** | 分页查询，支持关键词、分组筛选 |
| **收藏分组** | 获取用户创建的收藏分组列表 |
| **更新收藏** | 修改备注或分组 |
| **收藏数量** | 获取当前用户收藏总数 |

### 4.8 数据分析与统计模块

| 功能 | 说明 |
|------|------|
| **系统概览** | 专利总数、实体总数、领域总数等 |
| **实体词云** | 高频实体词汇及出现次数，用于词云图 |
| **实体类型分布** | 各实体类型数量统计 |
| **领域分布** | IPC 部级专利数量，用于饼图 |
| **专利趋势** | 按年份统计专利数量，用于折线图 |
| **申请人排行** | 专利申请数量 Top N |
| **缓存刷新** | 清除统计缓存，强制重新计算 |

## 6. 数据存储设计

### 6.1 MySQL 核心表

| 表名 | 用途 |
|------|------|
| sys_user | 系统用户 |
| patent | 专利基础信息 |
| patent_entity | 专利实体（LLM 提取） |
| patent_domain | 专利领域（IPC 层次） |
| patent_vector | 专利与 Qdrant 向量的映射 |
| patent_favorite | 专利收藏 |
| match_record | 匹配记录（专利匹配/文本查询） |

### 6.2 Elasticsearch 索引

- **索引名**：patent_index
- **主要字段**：id、publication_no、title、abstract、claims、applicant、apply_date、domain_codes、entities、parse_status
- **分词**：IK 分词器（ik_max_word / ik_smart）

### 6.3 Qdrant Collection

- **集合名**：patent_vectors
- **向量维度**：1536（通义千问 text-embedding-v3）
- **Payload**：patent_id、publication_no、title、domain_section、domain_codes、entities、entity_types 等

### 6.4 MongoDB 集合

| 集合 | 用途 |
|------|------|
| chat_sessions | 聊天会话（sessionId、userId、title、messageCount、status） |
| chat_messages | 聊天消息（sessionId、role、content、sequence、timestamp） |
